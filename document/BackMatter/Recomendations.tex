\begin{recomendations}
    
Más allá de los logros alcanzados, el proyecto deja abiertas varias líneas de evolución que permitirán al Instituto Finlay de Vacunas y a los desarrolladores que den continuidad al sistema llevarlo a su pleno potencial operativo. Se proponen las siguientes recomendaciones:

\begin{enumerate}
    \item \textbf{Migración a servidores físicos del Instituto Finlay.} Se recomienda al equipo de infraestructura del IFV trasladar el sistema desde los entornos de desarrollo (Railway y Vercel) a los servidores institucionales, utilizando la misma configuración de contenedores Docker y variables de entorno ya desarrollada. Esta migración garantizará el control sobre los datos de farmacovigilancia y eliminará la dependencia de plataformas externas.

    \item \textbf{Desarrollo de una aplicación móvil nativa.} Se recomienda al equipo de desarrollo evaluar la creación de una aplicación para iOS y Android que aproveche notificaciones push, acceso sin conexión a internet y una experiencia optimizada para pantallas táctiles. La API actual ya soporta esta evolución sin requerir modificaciones en el backend.

    \item \textbf{Ampliación de los canales de notificación.} Se recomienda migrar progresivamente los servicios de notificación de prototipado a producción: un servidor SMTP institucional para el correo electrónico y la API oficial de WhatsApp Business para la mensajería. Adicionalmente, se sugiere incorporar SMS como canal complementario para alcanzar a usuarios sin acceso a datos móviles.
\end{enumerate}

Estas recomendaciones se dirigen tanto al personal técnico del IFV encargado de la infraestructura y el desarrollo como a los directivos responsables de autorizar los contratos de servicios y la asignación de recursos necesarios para cada etapa.

\end{recomendations}
