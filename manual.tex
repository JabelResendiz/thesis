	\chapter{Introducción}
	
	El Sistema Web para el Reporte de ESAVI fue desarrollado para apoyar el proceso de registro, gestión y seguimiento de los Eventos Supuestamente Atribuibles a la Vacunación o Inmunización (ESAVI) en el Instituto Finlay de Vacunas.
	
	La plataforma permite centralizar la información relacionada con los reportes, facilitar la comunicación entre los diferentes actores involucrados y mejorar la trazabilidad de los datos registrados.
	
	Este manual tiene como objetivo orientar a los usuarios en el uso de las principales funcionalidades del sistema, proporcionando instrucciones básicas para el acceso, registro y consulta de la información.

Para registrar un nuevo reporte de ESAVI, seleccione la opción \textbf{Nuevo Reporte} desde la página principal del sistema.

La funcionalidad de inicio de sesión está destinada exclusivamente a los usuarios con permisos administrativos dentro del sistema, incluyendo médicos evaluadores, jefes de vacunación y administradores.

\subsection{Autenticación de usuarios}

\begin{figure}[htpb]

\section{Gestión de Duplicados}

Con el objetivo de evitar el procesamiento repetido de un mismo caso, el sistema identifica automáticamente reportes que podrían corresponder a un ESAVI previamente registrado.

Un reporte es marcado como \textbf{posible duplicado} cuando coincide con otro reporte existente en los siguientes criterios:

\begin{itemize}
\item Identificador del sujeto vacunado.
\item Fecha de vacunación.
\item Vacuna administrada.
\end{itemize}

La detección automática constituye únicamente una sospecha inicial. Corresponde al Jefe de Vacunación revisar la información disponible y determinar si ambos reportes representan realmente el mismo evento o si se trata de casos distintos.

\subsection{Acceso a la gestión de duplicados}

La funcionalidad de revisión de duplicados se encuentra disponible dentro del módulo \textbf{Gestionar Reportes}, mediante la opción \textbf{Ver Duplicados}.

\begin{figure}[htpb]
\centering
\includegraphics[width=0.8\textwidth]{jefe/ver_duplicados.drawio.png}
\caption{Acceso al módulo de gestión de duplicados}
\label{fig:ver-duplicados}
\end{figure}

\newpage

\subsection{Listado de posibles duplicados}

La Figura \ref{fig:listado-duplicados} muestra el listado de reportes identificados por el sistema como posibles duplicados.

\begin{figure}[htpb]
\centering
\includegraphics[width=0.8\textwidth]{jefe/duplicados.drawio.png}
\caption{Listado de posibles reportes duplicados}
\label{fig:listado-duplicados}
\end{figure}

Para cada caso se muestra información básica de ambos reportes, incluyendo:

\begin{itemize}
\item Nombre del sujeto vacunado.
\item Fecha del reporte.
\item Estado actual del reporte original.
\item Médico evaluador asignado, si existe.
\end{itemize}

Mientras no se emita un veredicto, el reporte sospechoso permanecerá bloqueado y no podrá ser asignado a ningún médico evaluador.

El Jefe de Vacunación dispone de las siguientes opciones:

\begin{itemize}
\item \textbf{Ver Comparación}: permite examinar detalladamente ambos reportes.
\item \textbf{Duplicado}: confirma que ambos reportes corresponden al mismo caso.
\item \textbf{No Duplicado}: indica que los reportes representan eventos diferentes.
\end{itemize}

\newpage

\subsection{Comparación de reportes}

\begin{figure}[htpb]

Entre los elementos que pueden analizarse se encuentran:

\newpage

\subsection{Marcación de un reporte como duplicado}

Si tras la revisión se concluye que ambos registros corresponden al mismo caso, debe seleccionarse la opción \textbf{Duplicado}.

\begin{figure}[htpb]
\centering
\includegraphics[width=0.8\textwidth]{jefe/mensaje_ver_reporte_duplicado.jpg}
\caption{Notificación de clasificación de reporte como duplicado}
\label{fig:mensaje-duplicado}
\end{figure}

Una vez procesada la operación, el sistema mostrará una notificación de confirmación en la esquina inferior derecha de la pantalla, similar a la mostrada en la Figura \ref{fig:mensaje-duplicado}. Este mensaje indica que la clasificación fue realizada correctamente.

Tras esta acción:

\begin{itemize}
\item El reporte original se mantiene activo.
\item El reporte identificado como duplicado es descartado.
\item El reporte descartado no podrá ser asignado a un médico evaluador.
\item La decisión quedará registrada en el sistema para futuras consultas.
\end{itemize}

Si se determina que ambos registros corresponden a eventos distintos, debe seleccionarse la opción \textbf{No Duplicado}.

\begin{figure}[htpb]
\centering
\includegraphics[width=0.8\textwidth]{jefe/mensaje_no_duplicado.jpg}
\caption{Notificación de clasificación de reporte como no duplicado}
\label{fig:mensaje-no-duplicado}
\end{figure}

Al completarse la operación, el sistema mostrará una notificación de confirmación en la esquina inferior derecha de la pantalla, similar a la presentada en la Figura \ref{fig:mensaje-no-duplicado}, indicando que la clasificación fue realizada exitosamente.

Tras esta decisión:

\begin{itemize}
\item El sistema considerará el reporte como un caso independiente.
\item El reporte podrá continuar su flujo normal de procesamiento.
\item El caso estará disponible para su asignación a un médico evaluador.
\item Ambos reportes permanecerán registrados como eventos independientes dentro del sistema.
\end{itemize}
